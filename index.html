CLCDSA阅读笔记

背景知识

Cross Language Code Clone （跨语言代码克隆）

  • 跨语言，即跨编程语言

  • 代码克隆，也称作重复代码或者相似代码，通俗的来讲，就是将别处具有当前所需功能的代码挪过来使用。我们平时在自己编写的程序中借鉴他人的代码就类似于这一过程。参见代码克隆检测研究进展.

    代码克隆(code clone), 也称为重复代码(duplicate code)或相似代码, 指的是存在于代码库中两个及以上的相同或者相似的源代码片段.代码克隆产生的原因有很多, 主要是开发者为了提高效率而使用的复用技术, 包括复制粘贴已有的代码片段并进行修改、使用开发框架、复用设计模式等.大量的实证研究表明, 代码克隆广泛存在于各个开源与闭源代码仓库之中, 并且占据了相当比例, 例如, 有研究在Linux系统中检测到了22.3%的代码克隆, Kamiya等人发现在JDK中存在29%的代码克隆, 在某些软件系统中代码克隆甚至达到了50%.广泛存在的代码克隆一定程度上帮助了软件系统的开发, 能够产生正面的效益, 比如可以利用克隆系统测试新增功能对原系统的影响, 然而也有许多研究指出数量巨大的代码克隆会对软件系统造成负面的影响:随着软件生命周期的进行, 未得到良好克隆管理的软件系统会因为代码克隆造成代码库的不断膨胀, 从而增加维护成本.软件缺陷也会因为代码克隆而在系统中被传播, 降低了软件系统的可靠性.所以, 如果不及时控制代码克隆的增长, 对系统的管理、维护、修复等行为都会耗费额外的人力, 导致软件维护成本的提高.

Abstract

  1. 代码克隆对software maintenance and evolution有害。
  2. 现存工具只能针对单一编程语言进行代码克隆检测。
  3. 有些程序可能由于移植性、不同平台的性能表现等原因对同一程序提供不同语言版本的克隆。
  4. 本文提供的CLCDSA通过分析不同编程语言之间的语法特征来检测代码克隆，为了提供对大规模克隆检测的支持，CLCDSA使用跨语言API（编程接口）调用的相似性来过滤非潜在克隆的样本。
  5. CLCDSA使用基于特征向量学习模型的DNN来学习特征和检测克隆，且不需要中间状态。
  6. 关键字 代码克隆 API文档 Word2Vector(词向量化) Source Code Syntax

Introduction

与现存工具的对比

单一语言代码克隆检测工具

CCFinder、Deckard、NiCad、SoucerCC

  • 这些工具仅支持同一编程语言下的代码克隆检测

多语言代码克隆检测工具

CLCMiner

只能检测不同编程语言环境下API名称或调用模式比较相似的代码克隆，对使用不同函数实现相同功能的代码克隆无能为力，这导致在实际的应用场景中CLCMiner很难应用。

LICCA

基于Budimac et al.'s SSQSA architecture提供的中间状态，SQSSA基于源代码提供增强具体语法树（eCST）。LICCA分析eCST并从中生成矩阵，之后将矩阵扩展到代码克隆检测。

然而LICCA具有诸多的限制：

  1. 要求源代码长度相同。

  2. 要求代码执行控制流相同。

这也导致LICCA在实际的应用场景中很难应用。

第一章 古典密码

密码学的重要性和意义

密码学是提供安全服务的关键理论和技术

  • 数据机密性（Confidentiality）
  • 数据完整性（Integrity）
  • 鉴别（Authentication）
  • 不可否认（Non—Repudiation）：数字签名
  • 其他：数字水印

基本术语和概念

基本术语

  • 按照明文的处理方法可分为分组密码（block cipher）和流密码（stream cipher）

  • 按照保密的条件可分为受限制的（Restriced）算法和基于密钥（key-based）的算法。

    • 受限制的算法：安全性基于算法的保密性。
    • Kerckhoffs假设：即使密码系统的任何细节已为人悉知，只要密匙（key，又称密钥或秘钥）未泄漏，它也应是安全的。这就是基于密钥的算法的原则之一。
    • 基于密钥的算法：安全性是基于密钥的安全性，而算法本身可以完全公开。基于密钥的算法通常有两类：对称密钥算法和公开密钥算法。

密码体制的定义

密码体制（密码系统）的数学描述

  • 它是一个五元组（P , C , K , E , D)，满足条件：

    • P是可能明文的有限集；（明文空间）

    • C是可能密文的有限集；（密文空间）

    • K是一切可能密钥构成的有限集；（密钥空间）

    • K是加密算法的有限集，D是解密算法的有限集

    • 对任意的k $ \in $ K，有一个加密规则（算法）$ e_k \in E $ 和相应的解密规则（算法）$ d_k \in D $ ，使得 $ e_k : P \rightarrow C $ 和 $ d_k : C \rightarrow P $ 分别为加密解密函数，满足 $ d_k(e_k(x))=x $ ,这里 $ x \in P $ 。

    • 注：加密函数必须是单射函数。

古典密码的实现技术

代换

加密：将明文字符按照对应关系代换成另外的密文字符

解密：将密文字符按照对应关系还原成明文字符

密钥：明密文字符之间的对应关系

种类：单表代换、多表代换、多字符代换

置换

加密：将明文字符按照一定的规则移动位置，得到排列错乱的密文，字符本身不变

解密：将密文字符按照响应的逆向规则还原成原来的顺序

密钥：移位规则

置换和代换的区别

置换可以理解为将明文按照某种规则进行打乱，而代换则是将每一个字符按照某种规则进行替换。

第一章 概述

冯诺依曼结构计算机工作原理及层次结构分析

冯诺依曼计算机的基本思想

存储程序和程序控制。

程序控制：按指令地址访问存储器并取出指令，经译码依次产生指令执行所需的控制信号，实现对计算的控制，完成指令的功能。

存储程序：将程序存放在计算机的存储器中。

冯诺依曼计算机的组成（硬件+软件）

硬件的三个主要部分，主机，外设，总线。

主机：CPU和内存。CPU由运算器和控制器构成，运算器即用于算数运算如加减乘除、逻辑运算如与或非移位，控制器用于产生指令执行过程所需要的所有控制信号，控制相关功能部件执行相应操作。内存用于存储源程序、原数据或运算中间结果。

外设：包括输入设备、输出设备和外存储器，输入输出设备用于外部数据的输入输出，如从键盘输入数据，向显示器输出数据。外存储器同样用于存储程序本身以及数据。

总线：包括地址线、数据线和控制线。

计算机系统性能评价

机器字长（非时间指标）

机器字长指机器一次能处理的二进制位数。

  • 由加法器、寄存器的位数决定。
  • 一般与内部寄存器的位数相等。
  • 字长越长，表示数据的范围就越大，精确度越高。
  • 目前常见的有32位和64位字长。

总线宽度（非时间指标）

总线宽度指数据总线一次能并行传送的最大信息的位数。

  • 一般指运算器与存储器之间的数据总线位数。
  • 有些计算机内部与外部数据总线宽度不一样。

主存容量与存储带宽（非时间指标）

主存容量指一台计算主存所包含的存储单元总数。

存储带宽指单位时间内与主存交换的二进制信息量，常用单位B/s（字节/秒）（影响存储带宽的指标包括数据位宽和数据传输速率）。

时间指标

主频f、时钟周期T

f与T互为倒数，f越高，T就越小（ f = 100 MHz 时 T = 10 ns ，f = 1 GHz 时 T = 1 ns）。

外频、倍频

外频指CPU（内存）与主板之间同步的时钟频率（系统总线的工作频率）。

倍频指CPU主频与外频之间的倍数。

  • 主频 = 外频 x 倍频

    如： Pentium 4 主频为2.4 GHz =2400 MHz = 133 MHz（外频） x 18 （倍频）

CPI(Clock cycles Per Instruction)

执行一条指令（平均）需要的时钟周期数（即T周期的个数）。

MIPS(Million Instructions Per Second)

该公式十分重要，一定要理解并牢记。

注意CPI的计算公式。

CPU时间

病毒的自我复制实验

需要实现的目标

  • 以C语言、汇编语言为开发工具，设计并开发一个简单的病毒，可在windows 7环境下，进行文件自我复制、感染的程序。
  • 可以完成多个目录下的多个拷贝。

  • 独自开发、独立运行。
我的实验过程记录
1.首先思考如何实现文件的感染

​ 在PE文件中，对函数以及内存变量的访问均基于偏移地址实现，而当病毒感染不同的HOST程序后，由于病毒代码依附到HOST程序中的位置各不相同，因此病毒随着不同HOST载入内存后，病毒中的各个变量（常量）在内存中的位置会随着HOST程序的大小不同而发生变化。

​ 而使用C语言编写程序时，可以看到如下图示例程序中，所有变量的偏移量由编译器在编译时确定，写死在机器码中。因此，无法使用C实现用于插入到其他可执行程序的代码，必须使用汇编实现。

但是我们可以将一段已经编译好的，具有在宿主程序中运行能力的代码的机器码复制出来，将其作为viruscode，然后在后续过程中使用C语言将其嵌入到目标程序中。

​

跳转到0433 执行指令E8 0000 而这正是我们viruscode的开始。

而程序入口的第一条指令为4D 然后是跳转指令E9 2F 03。

而这正是程序的第一行（这是COM文件的特点，没有多余的描述字段，直接在文件的开始启动跳转，并非标准的PE文件格式），因此只需设法使用C语言查找到程序的末尾的Offset，然后将其填入第3、4个字节以构造jmp指令即可。

根据jmp指令对应的机器码中提到的内容，当前IP为0101，而jmp指令本身占用3个字节，求得指令中填充的值表达式应为指令值=实际值-当前值-3。其中实际值为文件末尾的偏移量，但如果除去装载点的话，应当为在Winhex中获得的末尾位置333-1-3=32F，因此实现程序时，只需先获得文件末尾的偏移量，然后将其减去4后填充至被感染文件的第3、4个字节，然后将前两个字节也置为4D E9即可。

然而经过思考发现，只有在简单的、由几句汇编语言实现的PE文件中通过获取文件末尾的偏移量才易于实现，而在其他程序中较难实现，即使该程序只是简单的C编写的hello world程序。

而且经过思考，由于更改了文件起始的部分，那么无论把viruscode插入到哪里，都会导致原来的宿主程序不能正常运行，那么不如将viruscode放在Dos头刚刚结束的地方好了，这样的话可以将修改dos头部分的代码固定，而不是每次感染都需要重新查找和确定偏移。同时，这部分固定的代码可以成为我们的代码判断该程序是否已经被感染的标志，帮助我们实现不反复感染的功能。
